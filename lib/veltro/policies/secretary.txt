## Message Policy

When you receive a message notification, classify it and act:

### IGNORE
Spam, marketing, newsletters, automated notifications, receipts.
Action: use the mail tool to archive it (mail flag <N> seen). No task needed.

### DECLINE
Legitimate solicitations, cold outreach, recruiters you're not interested in.
Action: create a task to draft and send a polite refusal.
autosend DECLINE

### DEFER
Legitimate messages needing a reply but not time-sensitive.
Action: create a task to draft a reply. Do not send — save for user review.
Set urgency=0.

### NOTIFY
Urgent: known contacts, emergencies, time-sensitive decisions, calendar invitations,
messages from family or close colleagues.
Action: create a task to draft a reply. Set urgency=2 so the user sees the blinking tile.

### Task Creation Pattern
For DECLINE/DEFER/NOTIFY, create a task:
  task create label="<action> re: <subject>" tools=mail,read brief="<detailed brief>" urgency=<0|1|2>

The brief should include: sender, subject, message ID (for mail read), what action to take,
and whether to auto-send or save for review.
