editor - AI remote control for the Editor app

NOTE: This tool does NOT launch the editor. To start the editor,
use the launch tool: "launch editor". This tool sends commands to
an editor that is already visible in the presentation zone.

If the user says "launch the editor" or "open the editor", use
the launch tool, not this tool.

How it works: The Editor app exposes its state as files under
/tmp/veltro/editor/. This tool reads and writes those files.
The editor polls for commands every 500ms, so there is a brief
delay between sending a command and it taking effect.

Commands (only work after "launch editor"):
  read [body]              Read document body text
  read addr                Read cursor position (line col)
  write <text>             Replace entire document body
  append <text>            Append text to body
  save                     Save current file
  open <path>              Open file in editor
  goto <line>              Move cursor to line number
  find <string>            Search for text in document
  replace <find> <repl>    Replace next occurrence
  replaceall <find> <repl> Replace all occurrences
  addr                     Get cursor position (line col)
  insert <ln> <col> <text> Insert text at position
  delete <sl> <sc> <el> <ec>  Delete text range
  name <path>              Set file path without loading
  close                    Close editor
  status                   Show document info (id, path, dirty)

Typical workflow:
  launch editor                  1. Start the editor app
  editor open /usr/me/file.b     2. Open a file
  editor read                    3. Read the document body
  editor goto 42                 4. Jump to line 42
  editor find TODO               5. Search for text
  editor write <new content>     6. Replace body
  editor save                    7. Save to disk
